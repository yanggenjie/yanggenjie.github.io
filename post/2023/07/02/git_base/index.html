概念理解

要学好 git，主要是要理解 git 工作流程，git 分为三个阶段：工作区、暂存区、版本库。

  • 工作区：是你正在修改的文件；
  • 暂存区：是完成了一个小阶段的修改，但是还没完全修改完，也暂时不想添加到版本库的文件；
  • 版本库：是已经完全做完了，需要添加到版本库进行版本控制的文件。

Git 工作流程如下图，从左到右依次进行：

当你对一个文件进行修改保存之后，想要对文件进行版本控制，那就先把文件**从工作区添加到暂存区(git add)，然后再从暂存区添加到版本库(git commit)**。这个流程是按顺序进行的，不能直接从工作区直接添加到版本库。

这里使用SourceTree这种 git 可视化的工具来理解完成一次完整的版本控制，

1.Git 初始化

TestGit 文件夹下有一个 HelloWorld.txt 文件，想要对这个文件夹使用 Git 进行版本控制，就需要先进行初始化：

打开 SourceTree，点击 Create，然后选择路径，点击创建。用命令的话，就是git init。

初始化之后，文件夹下会多出.git文件，这个是保存所有版本历史的数据库，所有的版本数据都保存在这个文件夹里面。

2.添加到暂存区

重新回到 SourceTree，可以看到，当前工作区上有一个 HelloWord.txt 文件，

点击加号，添加到暂存区，相当于命令的git add

3.添加到版本库

然后写下一些说明，点击 commit 添加到版本库，相当于命令的git commit -m "说明"

版本记录完成，查看版本历史

这样一个版本就被记录下来了，之后不管做了多少次修改，只要.git文件夹不被删除，那就可随意回到某个版本。

后续：继续下一个版本的修改

后面继续更新内容的时候，就可以继续在原来的文件上进行修改


当修改了内容之后，文件会重新变成未暂存状态(unstaged)，

然后继续之前的流程：添加到暂存区，添加到版本库，版本历史就会多出新的记录

如果现在想查看第一个版本，只需要选中第一个版本的记录，然后检出即可;

但如果想在版本 1 之上上继续修改，那最好选择 Branch(分支)，即回到版本 1 的时候，使用一个分支来管理。


这样，文件夹内的文件就是版本 1 的内容：

这个就是基本的本地版本库的使用流程，但一般公司都会有自己的服务器，或者我们自己使用了 Gitee、github 之类的代码托管平台，就是相当于把本地的版本库和远程的版本库进行同步。

无论是把本地的文件推送到远程仓库，还是从远程仓库下载到本地，都是版本库和版本库之间的交互。

  • 本地推送到远程：git push
  • 远程拉取到本地：git clone或者git fetch，两者的区别是git clone是你本地什么都没有，完全要从远程下载。git fetch是本地已经有一个版本库，但可能落后于远程仓库，git fetch想当于同步；
  • 远程直接拉取到工作区：git pull

从下图就可以很清晰的看到这个流程，平时在本地使用比较频繁的操作就是添加到暂存区，添加到版本库，检出版本库，上传到远端的都是已经做好了的
