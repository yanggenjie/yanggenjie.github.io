conda 简介

在日常项目中，经常会面临同时管理多个项目，不同项目需要不同版本的软件包，如下面有三个项目，每个项目所需的 python 版本可能不一样，所需的库版本也不一样。

项目 python pkg1 pkg2
project1 3.9 V1 V1
project2 3.10 V1 V2
project3 3.10 V2 V2

而 Python 的版本比较多，并且它的库也非常广泛，同时库和库之间存在很多依赖关系，所以在库的安装和版本的管理上很麻烦。Conda 是一个管理版本和 Python 环境的工具，可以将各个应用环境隔离起来，互相不受影响。

安装 conda

目前可以安装 anaconda 和 miniconda，这两者之间的关系，就像 QQ 和 TIM 类似，QQ 提供了很多很强大的功能，而 TIM 只提供聊天、网盘文件等比较精简的核心功能，anaconda 和 miniconda 也有类似的关系，对于 anaconda，提供了 conda,python,以及其它一些科学计算的如 SciPy,NumPy 等等之类的众多工具，而 miniconda 只提供了一些比较核心的工具，比如 python,conda,以及它们所依赖的一些包，因此 miniconda 就是 anaconda 的一个精简版。

这里推荐安装 miniconda，后续有什么需要安装的，再使用 conda 来自己安装环境。

下载

官方下载：Miniconda miniconda documentation

清华大学开源软件镜像站：https://mirrors.tuna.tsinghua.edu.cn/anaconda/miniconda/

安装

  • windows

安装之前，确保电脑里面没有安装 python 环境，如果有，先卸载再安装，因为 miniconda 本身自带了一个 python,很有可能会和你电脑上的产生冲突。

下载 exe 文件，双击安装就行，注意使用默认的选项就行，不要添加到环境变量。

激活conda

  • windows
    安装完之后，就会有一个 Anacoda Prompt 的 Shell 程序，一个是 cmd 版本的，一个是 powershell 版本的。打开 cmd 版本，输入conda -h，看到下面这个界面就说明安装完成了。

  • Linux

更换源

由于 conda 默认安装源是在国外，安装各种软件包的时候可能会下载很慢，可以更换安装源为国内的镜像源，安装软件的时候会快很多。

无论是 windows、macOS 还是 Linux，更换源的方式都是修改家目录下的.condarc文件即可
